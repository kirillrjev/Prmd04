## Самостоятельная работа 2. Библиотека классов для учета сотрудников

## Цель

Разработать библиотеку классов для управления персоналом.

Применить принципы ООП: инкапсуляцию, наследование, полиморфизм.

Реализовать интерфейсы и абстрактные классы.

Создать систему с различными типами сотрудников и функционалом.

Организовать код в многослойной архитектуре.

Ход работы
## 1. Архитектура системы

Модели данных: классы сотрудников (FullTimeEmployee, PartTimeEmployee, Manager).

Интерфейсы: ISalaryCalculatable, IReportable, IPromotable, IVacationable, INotifiable.

Сервисы: EmployeeService для CRUD и бизнес-логики.

Утилиты: вспомогательные функции и константы.

## 2. Принципы проектирования (SOLID)

SRP: каждый класс решает одну задачу.

OCP: классы открыты для расширения, закрыты для модификации.

LSP: объекты базовых классов заменяемы объектами производных.

ISP: разделение интерфейсов по функционалу.

DIP: зависимости строятся через абстракции, а не конкретные реализации.


Основные компоненты
## 1. Интерфейсы

ISalaryCalculatable: расчет зарплаты, бонусов, налогов.

IReportable: генерация отчетов и summary.

IPromotable: проверка возможности повышения, повышение, бонусы.

IVacationable: управление отпускными днями.

INotifiable: уведомления по email/телефону.

## 2. Перечисления

EmployeeType: FullTime, PartTime, Contractor, Intern

Department: IT, HR, Finance, Marketing, Sales, Operations, Research

PerformanceRating: Excellent, Good, Average, BelowAverage, Poor

## 3. Абстрактный класс Employee

Свойства: Id, FirstName, LastName, Email, Phone, HireDate, Department, EmployeeType, PerformanceRating, BaseSalary.

Абстрактные методы: GetJobDescription(), CalculateSalary().

Виртуальные методы: DisplayBasicInfo(), SendNotification(), CanReceiveNotifications().

Валидация данных через свойства.

## 4. Конкретные классы

FullTimeEmployee: полноценная зарплата, бонусы, управление, отпуска, повышение.

PartTimeEmployee: зарплата по часам, меньшие бонусы, отпуска.

Manager: наследует FullTimeEmployee, управляет командой, имеет TeamBudget, генерирует отчеты.

## 5. Сервис EmployeeService

CRUD операции: Add, Get, Remove.

Расчет общей зарплаты.

Распределение годовых бонусов.

Генерация отчетов по отделам.

Получение сотрудников, подходящих для повышения.

Массовая отправка уведомлений.

## 6. Демонстрационная программа

Создает сервис и сотрудников.

Добавляет сотрудников и формирует команды.

Демонстрирует функционал: отображение информации, расчет зарплат и бонусов, управление отпусками, повышение сотрудников, массовые уведомления, командные отчеты.
